# 字幕校验：时间轴以原始字幕为准，只严格比对段数与编号

「时间轴一致」在 PRD 中语义模糊，落地必须精确，否则误报/漏报会直接损害「帮用户抓翻译错位」这一卖点。

决定校验分两级：

- **硬错误（阻断导出）**：段数与原始字幕不一致、编号不连续、非 UTF-8、SRT 语法非法。异常时定位到具体段落。
- **时间轴**：直接采用原始字幕的时间轴，译文字幕只贡献文本；因此不对译文时间戳做严格逐段比对（毫秒级漂移不报错）。

理由：真实场景中用户把原始字幕喂给外部 AI 翻译，AI 常重排毫秒或改动格式，但只要段数与编号对齐、文本按序回填到原始时间轴即可。最常见的翻车是漏段/多段/编号错乱——这些正是硬错误要抓的；而时间戳漂移无关紧要，严格比对只会制造噪音。
