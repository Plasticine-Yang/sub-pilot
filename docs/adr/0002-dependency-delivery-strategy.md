# 依赖交付：ffmpeg 随包，Whisper 模型分层

产品第一原则是「完全离线、开箱即用」，与 PRD 初稿「首启检测缺失再下载 ffmpeg」相矛盾，故调整依赖交付策略：

- **ffmpeg**：静态二进制随应用打包进 `resources/ffmpeg/`，装完即用、无需联网（+几十 MB，可接受）。
- **Whisper 模型**：分层交付。安装包内置 `base`（~142MB）作默认模型，保证首启即可离线转写；`small`/`medium`/`large-v3` 等在识别界面按需下载（带进度与校验）。

理由：模型体积悬殊（base 142MB 到 large-v3 3GB），全部打包不现实，全部按需又破坏首启离线体验；分层是两者的平衡。选 base 而非 PRD 默认的 small 作内置默认，是为了压低安装包体积，质量差距对多数视频可接受，用户可随时下载更大模型。
