# 项目状态持久化为目录 + project.json，不用数据库

一次字幕工作是有状态、可中断、可恢复的流水线（导入→转写→生成→翻译→导回→校验→导出），需要「项目」概念承载状态，还要支撑首页「最近项目」。

决定：每个 Project 是一个目录，内含源视频引用、`original.srt`、译文字幕与 `project.json`（记录状态机、所用模型、进度）。不引入 SQLite 等数据库。「最近项目」由应用数据目录下一个轻量索引维护。

理由：契合「完全本地、隐私友好、易扩展」原则——字幕产物是用户可见、可移植的普通文件，而非藏在 DB 里；避免文件与元数据分离导致的不一致，也免去数据库依赖。代价是跨项目查询需扫目录/索引，但项目数量级小，可接受。
